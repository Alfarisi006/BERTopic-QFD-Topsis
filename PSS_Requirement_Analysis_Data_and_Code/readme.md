Project title:
A Hybrid Approach to Analyse Product-Service System Requirement Integrating BERTopic, QFD, and TOPSIS

Purpose:
This repository provides the anonymized data, analysis code, intermediate outputs, and calculation files supporting the case study reported in the manuscript.

Main workflow:
1. Open-ended survey responses were cleaned and prepared.
2. BERTopic was applied to extract candidate requirement themes.
3. Extracted themes were interpreted and consolidated into Receiver State Parameters (RSPs).
4. FAHP was used to determine RSP weights.
5. QFD for PSS was used to translate RSPs into product and service characteristics and components.
6. TOPSIS was used to evaluate and rank three candidate PSS solutions.


Data note:
All survey responses have been anonymized. Personal identifiers and irrelevant private information have been removed.
