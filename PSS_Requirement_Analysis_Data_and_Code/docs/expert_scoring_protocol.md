# Expert Scoring Protocol

This file describes the expert-based judgement procedures used in the Receiver State Parameter (RSP) definition, FAHP weighting, QFD for PSS analysis, and TOPSIS solution evaluation stages.

## 1. Expert group for RSP definition and QFD for PSS analysis

After BERTopic-based theme extraction, a focus group was organized to interpret the extracted candidate themes and define the Receiver State Parameters (RSPs).

The focus group consisted of:

- 2 experienced professors
- 3 experienced researchers
- 3 postgraduate students
- 3 undergraduate students

Based on the candidate themes, representative keywords, and corresponding texts generated by BERTopic, the group discussed and refined the themes in the context of the campus shared wardrobe service. The candidate themes were then consolidated into five RSPs:

1. Reduction of Management Burden
2. Service Convenience
3. Perceived Hygiene
4. Cost-effectiveness
5. Style Variety

The same focus group also participated in the QFD for PSS analysis, including the assessment of relationships between RSPs and product/service characteristics, and between product/service characteristics and product/service components.

## 2. FAHP weighting of RSPs

The relative importance of the five RSPs was assessed using pairwise comparisons. A fuzzy preference judgment matrix was constructed using a 0.1–0.9 fuzzy preference scale.

The scale was interpreted as follows:

| Scale value | Interpretation |
|---|---|
| 0.9 | The row RSP is extremely more important than the column RSP |
| 0.8 | The row RSP is strongly more important than the column RSP |
| 0.7 | The row RSP is obviously more important than the column RSP |
| 0.6 | The row RSP is slightly more important than the column RSP |
| 0.5 | The row RSP and the column RSP are equally important |
| 0.4 | The row RSP is slightly less important than the column RSP |
| 0.3 | The row RSP is obviously less important than the column RSP |
| 0.2 | The row RSP is strongly less important than the column RSP |
| 0.1 | The row RSP is extremely less important than the column RSP |

The resulting judgments were processed using an FAHP calculation tool based on a fuzzy preference relation approach. A compatibility test was conducted. The compatibility index was 0.077, indicating that the judgments were acceptably consistent.

The normalized RSP weights used in the subsequent QFD for PSS analysis were:

| RSP | Receiver State Parameter | Weight |
|---|---|---|
| RSP1 | Reduction of Management Burden | 0.190 |
| RSP2 | Service Convenience | 0.165 |
| RSP3 | Perceived Hygiene | 0.240 |
| RSP4 | Cost-effectiveness | 0.190 |
| RSP5 | Style Variety | 0.215 |

## 3. QFD for PSS relationship scoring

The QFD for PSS analysis was conducted in two phases.

In Phase I, the weighted RSPs were mapped to product and service characteristics.  
In Phase II, the product and service characteristics were further mapped to product and service components.

In both phases, relationship strength was assessed using a 1–3–9 scale:

| Score | Interpretation |
|---|---|
| 1 | Weak relationship |
| 3 | Moderate relationship |
| 9 | Strong relationship |

The importance of each downstream element was calculated by multiplying the upstream element weight by the corresponding relationship strength and summing the results across all upstream elements.

In Phase I, the upstream elements were the RSPs, and the downstream elements were the product and service characteristics.

In Phase II, the upstream elements were the product and service characteristics, and the downstream elements were the product and service components.

## 4. TOPSIS evaluation of candidate solutions

Three candidate Product-Service System solutions were evaluated using TOPSIS:

1. Solution A: intelligent matching-oriented solution
2. Solution B: convenience and circulation-oriented solution
3. Solution C: quality assurance-oriented solution

Five experts participated in the TOPSIS evaluation:

- 1 campus manager
- 1 finance department manager
- 3 product designers with over 10 years of experience

The five RSPs were used as the evaluation criteria. Their weights were taken from the FAHP results. Since all five RSPs represented desirable receiver states, they were treated as benefit criteria.

Each expert evaluated the performance of each candidate solution with respect to each RSP using a five-point scale:

| Score | Interpretation |
|---|---|
| 1 | Very poor performance |
| 2 | Poor performance |
| 3 | Moderate performance |
| 4 | Good performance |
| 5 | Very good performance |

The individual expert scores were averaged to form the TOPSIS decision matrix. The decision matrix was then normalized and weighted using the FAHP-derived RSP weights. The positive ideal solution and negative ideal solution were identified, and the relative closeness of each candidate solution was calculated.

The final TOPSIS ranking was:

| Candidate solution | Relative closeness | Rank |
|---|---:|---:|
| Solution A | 0.4487 | 3 |
| Solution B | 0.5148 | 1 |
| Solution C | 0.4982 | 2 |

Solution B, the convenience and circulation-oriented solution, obtained the highest relative closeness and was therefore ranked first.
